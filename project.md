# 🧪 AGENT LAB: Master System Specification

<context>
You are an expert Principal AI/MLOps Engineer. We are building "Agent Lab" during a 48-hour hackathon.

Agent Lab is a local version control, CI/CD observability, and evaluation platform for LLM Agents. It allows developers to define their agent configurations via a YAML file, run automated evaluations against golden datasets, log multi-step execution traces, and view version diffs on a modern Next.js dashboard.

The goal is to build a "Vercel for Agents" experience that runs entirely locally, bridging the gap between agent development and robust MLOps infrastructure.
</context>

<architecture>
## Core Infrastructure
We are utilizing a dual-database architecture to ensure the CLI remains lightning-fast while still capturing deep observability data.

1.  **Observability Engine:** Local Langfuse (via Docker) to store massive, nested execution traces, latency metrics, and token costs.
2.  **State Engine:** Local SQLite (`.agentlab.db`) to store rapid CLI states, version history, and aggregated success rates.
3.  **Dynamic Runner:** A Python backend (`importlib` + `pyyaml`) that dynamically loads any user's LangGraph agent and executes it against a JSONL dataset.
4.  **Dashboard:** A Next.js front-end served via a FastAPI bridge to visualize the evaluation diffs.
    </architecture>

<implementation_steps>
Execute the build in the following distinct phases. Do not hallucinate external dependencies beyond the standard stack (LangGraph, Langfuse, FastAPI, Typer, Next.js).

### Phase 1: Infrastructure & State

1.  Generate the `docker-compose.yml` for Langfuse v3 and its Postgres dependency on port 3000.
2.  Create the `db.py` SQLite module to initialize `.agentlab.db` with a `runs` table (`id`, `version_tag`, `success_rate`, `avg_latency_ms`, `avg_cost_usd`, `timestamp`).

### Phase 2: The Core Runner Engine

1.  Implement `parser.py` to read the user's `agent-eval.yml`.
    - _Schema expects:_ `agent.entrypoint`, `dataset.path`, `dataset.input_key`, `dataset.expected_output_key`.
2.  Implement `runner.py`. This is the most critical backend component.
    - Use `importlib` to dynamically load the function string found in `agent.entrypoint`.
    - Loop through the `.jsonl` dataset.
    - **Crucial:** Initialize `langfuse.callback.CallbackHandler(tags=[f"version:{tag}"])` and inject it into the LangGraph execution config so traces stream to localhost:3000.
    - Calculate the dataset success rate and aggregate cost/latency from the Langfuse API, then save to SQLite.

### Phase 3: CLI & API Bridge

1.  Build `cli.py` using Typer.
    - `agentlab init` -> Scaffolds `agent-eval.yml`.
    - `agentlab eval --tag <v>` -> Triggers `runner.py`.
    - `agentlab ui` -> Spawns the FastAPI server and Next.js frontend.
2.  Build `server.py` using FastAPI.
    - `GET /api/versions` -> Returns SQLite run history.
    - `GET /api/diff/{v1}/{v2}` -> Calculates deltas between two versions and uses an LLM (gpt-4o-mini) to generate a brief, text-based summary of what behavior changed.

### Phase 4: Next.js Dashboard

1.  Initialize Next.js App Router with Tailwind and Shadcn UI (Dark mode).
2.  Build the **Overview Page**: A data table showing version history, paired with a Recharts line graph tracking `success_rate` over time.
3.  Build the **Diff Page**: A side-by-side layout comparing Version A and Version B. Display the LLM summary generated by the FastAPI backend at the top, followed by red/green metric delta indicators.
    </implementation_steps>
